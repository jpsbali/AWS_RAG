

**COMMUNITY PROJECT**

**Cloud Native RAG Solutions**

Building Production-Grade RAG on AWS, GCP & Azure

                                                                                 


**Duration:** 8 Weeks   |   **Teams:** 3 Groups of 10   |   **Format:** Remote & Collaborative

# **Project Overview**

This is a community-driven initiative where 30 students will collaborate to build production-grade Retrieval-Augmented Generation (RAG) solutions using cloud-native services. Three teams of 10 will each build the same RAG application on a different cloud platform, enabling a real-world comparison of AWS, GCP, and Azure for enterprise RAG workloads.

| Why This Project? Most RAG tutorials stop at local prototypes. This project bridges the gap to production by using real cloud infrastructure, team collaboration, and industry-standard evaluation. You’ll walk away with a portfolio-ready project and hands-on cloud experience. |
| :---- |

# **What You’ll Build**

Every team will build a complete, end-to-end RAG pipeline on their assigned cloud platform:

* **Document Ingestion Pipeline —** Upload, parse, and chunk documents using cloud storage (S3 / GCS / Azure Blob)

* **Embedding Generation —** Generate vector embeddings using cloud-native AI services

* **Vector Store —** Store and index embeddings (OpenSearch on AWS, Vertex AI on GCP, AI Search on Azure)

* **Retrieval & Generation API —** Build a FastAPI endpoint that retrieves context and generates answers

* **Frontend Application —** A simple Streamlit or web interface for users to interact with the system

* **Evaluation Pipeline —** Measure retrieval quality and generation accuracy using RAGAS framework
